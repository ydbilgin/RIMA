# Codex Review: Image #14 Diversity

## Executive Verdict

Image #14 is useful, but not as a new-class source. The 4x4 sheet is a mixed candidate board: some slots are canonical class fits, some repair Image #12 drift, and the bottom row contains strong skin/NPC material. The production answer is Yol 1 + Yol 2: limited class skin variants for active classes, plus a small Hub NPC cast for silhouettes that do not cleanly preserve class identity. Yol 3, the 11-12 class expansion, fails for v1 because it breaks the 10-class lock and expands skill, animation, weapon, UI, balance, and class identity cost.

- (4,2) blue/teal-haired character: do not make an 11th class. Best playable use is Frost Elementalist skin variant, if the silhouette keeps the Elementalist open-hand/rune-disc casting identity and the outfit is recolored toward dusty indigo, cream sash, and deep teal. Best non-playable use is Hub lore-keeper/alchemist.
- (4,4) dark-skin female character: best class variant target is Brawler if pose/hand wraps/guard are adjusted. Second-best is Ranger; third-best is Ravager. As described, it is too sleek/generic to be canonical without a state pass.
- Diversity expansion: MODIFY/PASS. It fits Karar #145 Class skin variants and does not violate #74/#100/#144 if skins are cosmetic, body-only, 64x64 chibi, high top-down 30-35 degrees, and do not add mechanics.

## Q2: (4,2) Blue/Teal-Haired Character

The only safe playable rebrand is Frost Elementalist skin. Elementalist already owns elemental variance, hand-cast VFX, and rune-disc runtime augmentation. A frost skin can change hair and secondary palette while keeping the same mechanics. Required constraints:

- Hair may be teal/blue, but silhouette must still read as Elementalist: upright female caster, open palm, rune-disc/VFX hook.
- Outfit should not remain generic lavender. It should move toward dusty indigo plus cream sash plus deep teal skirt so the class family remains legible.
- Frost is cosmetic language only. No ice skill tree, no new damage type requirement, no alternate passive.

As an NPC, (4,2) is also strong: Hub alchemist, rift archivist, or lore-keeper. This fits Cinematic Layer v1 because it already supports Hub NPC mini-loops and small post-run reveals. Safest path: NPC first, Elementalist Frost skin after canonical Elementalist passes.

Making (4,2) an 11th Frost/Ice Mage class is a scope violation for v1. It creates new skills, VFX identity, balance curves, class select UI, animation prompts, weapon/VFX rules, and narrative onboarding. That is not a skin; it is a class expansion.

## Q3: (4,4) Dark-Skin Female Character

(4,4) can be integrated, but the current description lacks a class-defining hook. The best target is Brawler dark-skin female variant, because Brawler already canonically owns dark skin and bare-fisted identity. Required changes: short hair can stay, but hands need wraps, pose needs guard or forward fight stance, and outfit should be simple athletic wrap rather than sleek assassin clothing.

Ranger is the second-best option if the goal is broader class diversity beyond Brawler. To work as Ranger, (4,4) needs forest tones, quiver/bow-ready body language, and preferably a hair/cloak silhouette that is not confused with Shadowblade. Ravager is third-best: a dark-skin female Ravager would be strong, but needs dark blood-red armor, leather harness, studs, broader stance, and a more brutal posture. As-is, "sleek dark outfit" reads closer to assassin, which risks inventing an 11th rogue class and overlapping Shadowblade.

Recommendation: use (4,4) as Brawler Female Variant A for the first diversity pilot. If Brawler already has (2,2) canonical male and (4,1) monk alt, then (4,4) can become Ranger dark-skin female Variant A, but only after Ranger bow/quiver identity is added.

## Q4: Skin Variant System Verdict

Verdict: MODIFY/PASS.

The system is compatible if it stays cosmetic. Karar #74 is preserved by 64x64 and PPU=64. Karar #100 is preserved by chibi 3-4 head proportion and high top-down 30-35 degree camera. Karar #144 is preserved because skins are body sprites only; weapons remain WeaponSR child sprites or runtime VFX. Karar #145 explicitly supports Class skin variants. Karar #143-E is mostly orthogonal; Hub/NPC scene integration must still respect sorting layer discipline.

Do not launch with 2-3 variants for all 10 classes immediately. That is 20-30 extra body identity targets plus cleanup and QC. Use a narrow pilot:

- Batch 1: 3 variants: Elementalist Frost (4,2), Brawler Female (4,4), Brawler Monk or Ranger Dark-Skin Female (4,1 or reworked 4,4).
- Batch 2: 4 variants after pass: Ravager Female (3,2), Ranger Alt (1,2 or 4,4 reworked), Warblade Beard Repair if (1,1) passes, Hexer Hood Alt (2,1).
- Full system later: 1 alt skin per class first, then only high-value second skins.

PixelLab Character States cost estimate: if canonical bases are approved, each skin is roughly one Character State variant pass plus cleanup/QC. Minimal pilot = 3-4 state batches. One alt per 10 classes = 10 state batches. Two alts per 10 classes = 20 state batches. Because #145 runs through Web UI V3/manual workflow, the real cost is prompt discipline, direction consistency, cleanup, and Unity import naming.

Sprint target: after Warblade/Ranger/Shadowblade/Elementalist pilot stability, run a "Skin Pilot Sprint" with 3 variants max, no mechanics, no class-select redesign beyond locked preview slots.

## Q5: NPC Hub Characters Verdict

Verdict: PASS with tight scope.

Hub NPCs are a good home for strong but class-unsafe silhouettes: (4,2), (3,3), (4,3), and optionally (4,1). Cinematic Layer v1 already names Hub NPC mini-loops and run-after-run one-line progression, so these characters can serve narrative without expanding combat scope.

Recommended NPC count for v1: 3.

- Vendor/crafter: handles upgrades, relic cleanup, or weapon skin preview.
- Mentor/trainer: old silver-haired figure from (3,3) or (4,3), gives class tips or post-run failure lines.
- Lore-keeper/alchemist: (4,2), explains rift/weather/elemental anomalies.

Do not add more than 3 Hub NPCs in v1. More NPCs create writing, namecard, animation, and localization burden. If (4,1) is not used as Brawler Monk skin, make it the trainer and merge the silver-haired mentor into lore/background.

## Q6: 11-12 Class Expansion Verdict

Verdict: FAIL for v1.

Path C can exist only as post-v1 expansion planning. It should not justify Frost Mage or dark-skin Assassin now. A new class requires skills, class select UX, weapon/VFX identity, balance, passives, progression, animation states, and narrative justification. The project is still stabilizing canonical 10-class identity, weaponless body workflow, and Character States production. Adding classes before those pass creates drift and production debt.

If preserved, Path C should be written as "Expansion Backlog: candidate archetypes only" with no v1 commitments.

## Q7 Final Diversity Strategy

Use a combination of Yol 1 and Yol 2.

Target numbers:

- v1 playable skins: 3 pilot variants first, then expand to 10 total alt skins only after QC pass.
- v1 Hub NPCs: 3 total, using the strongest non-class silhouettes.
- v1 new classes: 0.

Sprint order:

1. Lock canonical 10-class base identity first. Image #14 can help repair Warblade and Ravager, but do not let variants mask unfinished canonical drift.
2. Run Skin Pilot Sprint with Elementalist Frost (4,2), Brawler Female (4,4), and either Brawler Monk (4,1) or Ranger Dark-Skin Female.
3. Run Hub NPC Sprint with lore-keeper, mentor/trainer, and vendor/crafter. Use Cinematic Layer v1 one-line progression, not full dialogue trees.
4. Only after 10 canonical bases and 3 pilot skins pass visual QC, decide whether every class gets exactly one alt skin.

Final call: diversity expansion is strategically correct if it is treated as cosmetic skin breadth plus a small Hub cast. It becomes harmful if it is used to reopen class count, skill count, or core identity locks.
