# AX (Gemini) RESEARCH — "Press-and-hold + drag" continuous-action UX pattern

Pure research/survey task (NO media download). You are a game-UX researcher.

## TOPIC
A factory/automation game (Conor Dart's, "Gourdlet"-like) lets the player place transport belts by **holding the mouse button and dragging** — the belt auto-routes along the drag path with auto-orienting corners. Survey this "press-and-hold + drag → continuous/auto placement" interaction family.

## RESEARCH & ANSWER (concise, practical)
1. **Survey** games that use hold+drag continuous placement and what they do well: Factorio (belt drag + auto-corner), Satisfactory, Mindustry, Dyson Sphere Program, Shapez, Timberborn, Townscaper (click-paint auto-WFC), Stardew Valley / farming sims (drag-till / drag-plant / drag-water rows), city builders (drag zoning/roads), mobile builders. For each: what exactly is held-and-dragged, and the key UX trick.
2. **Core UX rules** that make hold-to-build feel good: ghost/preview while dragging, auto-routing & auto-orientation (corners), auto-tiling (Wang) for connected pieces, snapping to grid, live cost/resource feedback, cancel/undo (right-click), drag straight-line vs free-path modes, "paint vs line" modes, release-to-commit.
3. **Pitfalls**: accidental over-placement, fighting the auto-router, no undo, unclear cost, performance on long drags.
4. **Two contexts**: (a) how this pattern fits a COZY BUILDER/FARMING sim (lots of repeated tile actions), (b) how a hold/charge interaction translates to a TOP-DOWN ACTION game (charge attacks, channeled beams, hold-to-aim/auto-fire, hold-to-interact) — the action genre rarely "drag-builds" but uses "hold-to-charge/channel".
5. Bullet list of concrete dos/don'ts.

Keep it a focused reference (no fluff). Output markdown.
