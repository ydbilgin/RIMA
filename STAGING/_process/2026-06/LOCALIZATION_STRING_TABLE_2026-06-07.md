## Uygulama Önerisi

### 1. Sistem Tasarımı ve Yoğunluk
- **Toplam String Yoğunluğu:** UI üzerinde yaklaşık **70-85 adet** metin tespit edilmiştir. Bu metinler özellikle `CharacterSelectScreen`, `SettingsMenuUI`, `SkillOfferUI` ve oyun sonu (`DeathScreen`/`VictoryOverlay`) ekranlarında yoğunlaşmaktadır.
- **Veri Yapısı:** `LocalizationService` içerisinde basit bir dictionary mantığı ile `Dictionary<string, string>` olarak tutulabilir. `System.Language` veya özel bir `enum Language { TR, EN }` kullanılabilir. JSON veya ScriptableObject ile veriler kolayca okunabilir.

### 2. Dil Ayarı Entegrasyonu (SettingsMenuUI)
- `SettingsMenuUI` içine **GAMEPLAY (OYNANIŞ)** başlığı altında en üste "Language / Dil" ayarı eklenmelidir.
- Bu ayar için `PlayerPrefs` anahtarı olarak `"setting_language"` (örneğin 0 = EN, 1 = TR) önerilir.
- Ayar değiştiğinde `LocalizationService.Instance.SetLanguage()` tetiklenmeli ve tüm aktif UI'lara `Action OnLanguageChanged` event'i fırlatılarak panellerin kendini canlı güncellemesi sağlanmalıdır.

### 3. Kullanım Önerisi
Kodlarda atamalar şu formata dönüştürülmelidir:
```csharp
titleLabel.text = LocalizationService.Get("draft.title_generic");
```

Formatlama gerektiren metinler (`string.Format`) için:
```csharp
deathStats.text = string.Format(LocalizationService.Get("death.stats.total_echo"), echoAward);
```

Dinamik Action promptları (`[G] Rift'e Gir`, `[RMB] İnfaz` vs.) için `HUDController` veya `ExecutePromptDriver` içerisinde doğrudan locale çağrıları yapılarak UI standartlaştırması garanti altına alınmalıdır.
