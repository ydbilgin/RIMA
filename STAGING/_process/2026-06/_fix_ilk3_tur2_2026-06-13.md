## FIX 2 — TelemetryClock aktif pause aralığını düşürür (pause'da DONAR)

**Kök neden (tur 1 neden yetersizdi):** `TelemetryClock() = Time.unscaledTime - telemetryPausedTotal`; aktif (devam eden) pause aralığı ancak çıkışta `telemetryPausedTotal`'a ekleniyordu. Director açıkken gelen telemetri kayıtları (`OnDamageAppliedTelemetry` 2206, `OnTelemetryTargetDeath` 2247) pause sırasında ilerleyen timestamp alıyordu.

**Değişiklik (`DirectorMode.cs:2378-2386`, sadece TelemetryClock fonksiyonu):**
```
activePause = (State==Director && pauseStart>=0) ? unscaledTime - pauseStart : 0
return unscaledTime - telemetryPausedTotal - activePause
```
Çağrı yerleri değişmedi. `ClearTelemetry` rebase mantığıyla (2275-2276: total=0, pauseStart= pause'daysa unscaledTime) tutarlı: clear anında activePause=0 → saat o anki bazda donar.

**Data-proof (execute_code, reflection):**
- `FREEZE_FORMULA_PASS=True` — pause'da clock = `pauseStart - total`; `unscaledTime` terimi TAM iptal → saat zamandan bağımsız → DONUK.
- `LIVE_PASS=True` — Test state'te activePause kapalı → clock = `unscaledTime - total`, zamanla ilerler.
- `CONTINUITY_PASS=True` — pause çıkışında (elapsed → total, pauseStart=-1) saat son pause değerine eşit, sıçrama yok.
