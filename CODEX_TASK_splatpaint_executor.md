## Goal

Add a paint brush executor to RIMA Map Designer that writes directly to the **splat map** consumed by `RIMA/TerrainBlend` shader. User drags mouse on the terrain mesh → splat texture gets painted in the selected channel (R/G/B/A = stone/path/moss/grass) → shader re-blends materials live in editor.

This is the "Aseprite for splat" tool. It replaces the procedural splat map currently generated by `TerrainBlendRenderer.Render(RoomData, ...)`. Users hand-paint zones.

## Functional requirements

### SplatPainterWindow (EditorWindow)

Menu: `RIMA → Map Designer → Splat Painter`

UI (use IMGUI, NOT UI Toolkit, for editor consistency):
- Target field: `MeshRenderer` (auto-find first object with `RIMA/TerrainBlend` shader in active scene)
- Splat texture preview (current splat at 256×192 preview size, RGBA visualized as 4 small thumbnails OR composite)
- Channel selector: 4 radio buttons (R = Stone, G = Path, B = Moss, A = Grass) with material name labels pulled from the bound TerrainBlendConfig texture names if available
- Brush radius slider: int, 1-30 splat-pixels (default 6)
- Brush opacity slider: 0..1 (default 1.0 — full overwrite)
- Brush hardness slider: 0..1 (default 0.7 — 1.0 = hard edge, 0.0 = full feather)
- Splat resolution selector: 64×48 / 128×96 / 256×192 (default 128×96)
- Buttons:
  - `Clear Channel` — fills selected channel with 0
  - `Fill Channel` — fills selected channel with 1
  - `Save as Asset` — writes splat to `Assets/Art/TerrainBlend/splat_<scenename>.png`
  - `Load Asset` — picks a saved splat
- Status line: current splat dimensions, dirty/clean flag

### Painting interaction

When window is focused AND user clicks in Scene View on the target mesh:
1. Raycast mouse → mesh world position
2. Convert world pos → mesh local UV [0..1] (mesh is a quad in TerrainBlendRenderer)
3. UV → splat pixel coord (clamp to bounds)
4. Apply circular brush: for each splat pixel within `radius`, compute distance `d`, weight = smoothstep(radius, radius * hardness, d) * opacity
5. Write to selected channel: `channel = lerp(channel, 1.0, weight)` (additive paint)
6. Other channels: `other = lerp(other, 0.0, weight)` (so painting Stone removes Grass underneath — channels are mutually exclusive at any pixel)
7. After mouse-up: `texture.Apply()` and `mat.SetTexture("_SplatMap", tex)`

Drag = continuous paint (stamp every frame while button held).
Right-click drag = erase (paint with channel = 0, others unchanged).
Shift+click = larger brush preview (no functional change, just visual).

Use `SceneView.duringSceneGui` callback for input handling. Repaint the SceneView on drag.

Write at minimum 3 tests in `SplatPaintExecutorTests.cs`:
